# Site Template 第一阶段规格说明（2026-04-19）

## 1. 背景

NoCode 当前主线已经从更早的 HTML 流式生成落盘排查，切换到 `site_template` 第一阶段能力收口。

本文件用于把这轮已存在的真实实现，收敛成一份可回看的专题规格说明，避免后续会话继续被旧问题上下文带偏。

---

## 2. 当前阶段目标

第一阶段目标是完成模板能力的最小业务闭环，让用户可以：

1. 查看模板
2. 基于模板创建应用
3. 将已有应用保存为模板
4. 编辑自己创建的模板
5. 让管理员控制模板公开状态

本阶段重点是先跑通基础闭环，不追求一次覆盖完整运营能力。

---

## 3. 当前已实现能力

基于当前仓库代码核对，已确认存在以下后端能力：

### 3.1 模板实体与基础数据结构
- 已存在 `SiteTemplate` 实体
- 已存在模板相关 DTO / VO
- 应用与模板能力之间已经建立基础字段关联

### 3.2 模板查询能力
- 支持模板详情查询
- 支持模板分页列表查询
- 模板列表已加入权限 / 可见性控制收口

### 3.3 模板创建能力
- 支持直接新增模板
- 支持从现有应用创建模板

### 3.4 模板更新能力
- 支持模板拥有者更新模板基础信息
- 支持管理员更新模板公开状态

### 3.5 模板删除能力
- 支持模板拥有者删除自己的模板
- 支持管理员删除模板

---

- `/template/add`
